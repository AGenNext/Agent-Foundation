# Federated Agreement Model

Agent-Foundation is open through federated agreements.

The platform cannot enforce behavior across the world. It can recommend, certify, recognize, publish compatibility signals, and decide what becomes trusted inside the AGenNext ecosystem.

## Core rule

```text
Open participation. Federated agreement. Recommended practice. Guarded trust.
```

## Definition

A federated agreement is a visible, reviewable, scoped agreement that lets individuals, communities, repositories, partners, providers, and organizations participate in the ecosystem through shared expectations.

The platform does not control external participants. It defines what it recognizes as trusted, compatible, supported, or recommended.

```text
Participant
  -> Agreement
  -> Scope
  -> Commitment
  -> Evidence
  -> Contribution
  -> Review
  -> Recognition
```

## What the platform can do

The platform can:

```text
recommend practices
publish compatibility criteria
certify integrations
recognize trusted artifacts
accept or reject contributions
label supported implementations
publish reference architectures
publish warnings or deprecations
refuse to promote untrusted work
```

## What the platform cannot do

The platform cannot globally enforce:

```text
how every external party behaves
what every fork does
what every vendor ships
what every community adopts
what every deployment runs
```

## Recognition levels

```text
Proposed
Compatible
Recommended
Certified
Supported
Deprecated
Untrusted
```

## Work agreement object

Every significant workstream should be represented as a GitHub issue, proposal, or project item.

A work agreement should define:

```text
participant
scope
repository
objective
affected contracts
risk level
commitments
evidence required
reviewers
acceptance criteria
recognition path
rollback or recovery path
```

## Agreement lifecycle

```text
proposed
  -> reviewed
  -> accepted
  -> in-progress
  -> validated
  -> recognized
  -> monitored
  -> deprecated or archived
```

## Federation boundaries

Participants may:

```text
open issues
start discussions
submit pull requests
propose vocabulary
propose contract changes
build SDK examples
build integrations
publish compatible implementations
```

Participants may not be marked trusted by AGenNext unless they satisfy the recognition criteria.

## One-line definition

Federated agreement means AGenNext cannot enforce the world, but it can recommend practices, define compatibility, recognize trusted work, and protect what becomes trusted inside its own ecosystem.
