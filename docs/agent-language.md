# Agent Language

The Agent Language defines the canonical DSL, vocabulary, and operational semantics for all Agents in the AGenNext Fabric ecosystem.

## Core principles

1. **Declarative**: Agents describe desired outcomes, constraints, and observations.
2. **Composable**: Agents can combine other Agents, skills, or tools.
3. **Observable**: All outputs, decisions, and state transitions are recorded with provenance.
4. **Reconciliable**: Agents propose changes; Fabric validates and reconciles.
5. **Contextual**: Agents understand environment, runtime, and operational constraints.
6. **Versioned**: Every Agent behavior, skill, and memory type is versioned.
7. **Human-accountable**: Decisions or proposals that change trusted state require human gate approval.

## Syntax

Agent Language consists of:

```text
ENTITY       : defines data objects or actors
ACTION       : defines operations or skills
CONDITION    : pre/post constraints or guards
RULE         : governing logic combining entities/actions/conditions
EVENT        : triggers observable outcomes
MEMORY       : persistent state or context
PROVENANCE   : metadata linking action to source
```

Example:

```text
ENTITY Agent "SurrealDB" {
    ACTION storeEvent(event)
    ACTION query(statement)
    CONDITION mustNotAlterFabric
    MEMORY eventLog
    PROVENANCE linkedToCommit
}

RULE reconcileProjection {
    WHEN eventLog changes
    THEN proposeUpdateToFabricRuntime
}
```

## Execution semantics

- Agents **never alter Fabric directly**.
- Agents **record evidence** of all operations.
- Fabric Runtime **reconciles** proposed changes to the canonical state.
- Human accountability gates ensure critical decisions are approved.

## Vocabulary

- **Agent**: autonomous or semi-autonomous actor in Fabric.
- **Skill**: reusable capability an Agent may invoke.
- **Tool**: external service or library callable by an Agent.
- **Memory**: persistent or ephemeral context state.
- **Event**: observable trigger or output.
- **Constraint**: rules or conditions that must be respected.
- **Decision**: outcome generated by Agent actions.

## Integration

Agent Language is implemented in:

- TypeScript DSL for Agent Builder and IDE
- JSON/JSON-LD schema for contracts and provenance
- Rust runtime adapters for Fabric Runtime

## One-line definition

Agent Language is the declarative, composable, observable, and reconciliable specification for Agents to act, propose, and interact safely within the AGenNext Fabric ecosystem, with human-accountable governance.
